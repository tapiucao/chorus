## Problema atual
- a API da Anthropic está recebendo o nome do modelo com quebra de linha e espaços
- erro observado:

```text
model:
  claude-sonnet-4-6
```

- isso acontece porque os `export CHORUS_MODEL_*` foram feitos quebrando a string em duas linhas no shell
- enquanto isso não for corrigido, a Anthropic responde `not_found_error`

## Comandos corretos
Executar no mesmo terminal onde o servidor será iniciado:

```bash
cd /home/tarun/.openclaw/workspace/projects/chorus
source venv/bin/activate

unset CHORUS_MODEL_EXTRACTION
unset CHORUS_MODEL_SYNTHESIS
unset CHORUS_MODEL_CRITIC

export ANTHROPIC_API_KEY='...'
export CHORUS_MODEL_EXTRACTION=anthropic/claude-sonnet-4-6
export CHORUS_MODEL_SYNTHESIS=anthropic/claude-sonnet-4-6
export CHORUS_MODEL_CRITIC=anthropic/claude-sonnet-4-6
export CHORUS_MODEL_EXTRACTION_FALLBACKS=
export CHORUS_MODEL_SYNTHESIS_FALLBACKS=
export CHORUS_MODEL_CRITIC_FALLBACKS=
export CHORUS_LLM_TIMEOUT_SECONDS=180

printf '<%s>\n' "$CHORUS_MODEL_EXTRACTION"
printf '<%s>\n' "$CHORUS_MODEL_SYNTHESIS"
printf '<%s>\n' "$CHORUS_MODEL_CRITIC"

./venv/bin/python -m uvicorn web.app:app --reload
```

Saida esperada do `printf`:

```text
<anthropic/claude-sonnet-4-6>
<anthropic/claude-sonnet-4-6>
<anthropic/claude-sonnet-4-6>
```
